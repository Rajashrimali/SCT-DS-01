# Contribution Guidelines

Thank you for your interest in contributing to this project! Please follow the guidelines below to ensure a smooth and effective contribution process.

## How to Fork
1. Click on the **Fork** button on the top right corner of the repository.
2. This creates a copy of the repository in your GitHub account.

## Create Branches
1. Navigate to your cloned repository in your local development environment.
2. Use the following command to create a new branch:
   ```bash
   git checkout -b <your-branch-name>
   ```
   - Example: `git checkout -b feature/my-feature`

## Make Commits
1. Make changes to the files in your branch.
2. Add the changes to staging:
   ```bash
   git add <file-name>
   ```
   - To add all changes: `git add .`
3. Commit your changes with a clear message:
   ```bash
   git commit -m "Your commit message here"
   ```

## Submit Pull Requests
1. Push your changes to your forked repository:
   ```bash
   git push origin <your-branch-name>
   ```
2. Go to the original repository on GitHub where you want to submit the pull request.
3. Click on the **Pull Requests** tab, then click on **New Pull Request**.
4. Select your branch from the drop-down menu.
5. Provide a title and description for your pull request.
6. Click on **Create Pull Request** to submit.

## Additional Notes
- Ensure that your code adheres to the project's coding standards.
- Always keep your branches up to date with the main branch.

Thank you for contributing!
